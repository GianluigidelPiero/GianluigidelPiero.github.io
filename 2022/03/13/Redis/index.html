Redis

数据结构

Strings

最简单Redis类型。如果只用这种类型，Redis就像一个可以持久化的memcached服务器（注：memcache的数据仅保存在内存中，服务器重启后，数据将丢失）。

原子递增 incr

将字符串值解析成整型，将其加一，最后将结果保存为新的字符串值。

即使多个客户端对同一个key发出INCR命令，也决不会导致竞争的情况。

mset mget

为减少等待时间，也可以一次存储或获取多个key对应的值。

exists del type

exists 判断key是否存在

del 删除key的值

type 返回key对应的值的存储类型

设置超时时间

两种方法

persist 去除超时时间

ttl 查询剩余的时间

Lists

Redis lists基于Linked Lists实现。这意味着即使在一个list中有数百万个元素，在头部或尾部添加一个元素的操作，其时间复杂度也是常数级别的。

缺点：访问速度慢

Redis Lists用linked list实现的原因是：对于数据库系统来说，至关重要的特性是：能非常快的在很大的列表上添加元素。

添加

从左边添加和从右边添加

删除

从左边删除和从右边删除，返回删除的值

遍历查询

lrange从左边开始查询

-1代表最后一个元素

截取

ltrim从左边开始截取

阻塞

list是空时，消费者需要轮询来获取数据，这样就会增加redis的访问压力、增加消费端的cpu时间，而很多访问都是无用的。

brpop blpop:
当list包含元素的时候，表现得跟 rpop和lpop一样。当list是空时，Redis将会阻塞这个连接，直到另一个客户端 push 元素进入或者达到timeout时限。

自动创建与删除

最后一个元素被删除后，这个key自动销毁

应用

聊天软件，不同进程间传递消息的队列。

Hashes

Hash特别便于表示对象，hash的域表示对象的属性。

HMSET 指令设置 hash 中的多个域。

HGET 取回单个域。

user:100的形式类似键值对。

Sets

Redis Set 是 String 的无序排列。

Sets 适合用于表示对象间的关系。

增 sadd

新闻 ID 1000 被打上了 1,2,5 和 77 四个标签，我们可以使用一个 set 把 tag ID 和新闻条目关联起来

查

所有 smember

某个 sismember

事务

MULTI 执行之后， 客户端可以继续向服务器发送任意多条命令， 这些命令不会立即被执行， 而是被放到一个队列中， 当 EXEC命令被调用时， 所有队列中的命令才会被执行。

另一方面， 通过调用 DISCARD ， 客户端可以清空事务队列， 并放弃执行事务。
